                                                                                                                                Empty state

                                                                                                                                Stable

                                                                                                                                Empty states occur when there isn’t any data or content to show. They can appear when it’s the first time a user encounters an app feature, after successfully clearing a queue of tasks, when there’s an error, or when a search returns no results. The purpose of an empty state message is to keep the user moving confidently forward, often with a fuller understanding of how the app works.
